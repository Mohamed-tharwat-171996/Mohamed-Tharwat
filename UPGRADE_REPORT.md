# 📋 دليـل الترقية والترحيل للمجموعـة (Upgrade & Migration Report)

بناءً على التوجيهات الحريصة والدقيقة، تم بنجاح استبدال نظام الملف البسيط `db.json` بنظام **قاعدة بيانات SQLite** خادم-جانب (Server-Side) متكامل ومستقل، مع توفير حماية أمنية مشددة تعتمد على تشفير **Bcrypt** لعمليات التحقق وتطبيق بروتوكولات **JWT Authentication** المحصنة.

---

## 🛠️ 1. هيكلية الملفات المحدّثة والجديدة (File Structure Split)

تم تقسيم وتوزيع الكود من ملف خادم أحادي إلى طبقات منفصلة متناغمة وبمستوى مؤسساتي كالتالي:

*   **`/.env.example`** *(جديد)*: يضم التصريح بمتغيرات البيئة مثل رمز التوقيع الرقمي للرموز الآمنة `JWT_SECRET`.
*   **`/server/database/dbService.ts`** *(جديد)*: يمثّل قاعدة البيانات والمحرك المنظم لتهيئة الجداول وعمل الترحيل التلقائي من `db.json` مع تأمين المعاملات المانعة للتلف (SQLite Transactions).
*   **`/server/services/auditService.ts`** *(جديد)*: وحدة تسجيل الأنشطة الحساسة لتوثيق التحركات الأمنية وجلسات الاسترداد ودخول المشرفين.
*   **`/server/services/sessionService.ts`** *(جديد)*: وحدة إدارة الجرد الحالي، الأرشفة، والترشيح الدائري التلقائي للاحتفاظ بالـ 39 نسخة الأخيرة وحذف الأقدم تلقائيًا.
*   **`/server/auth/authService.ts`** *(جديد)*: وحدة التخمير والتأمين عبر Bcrypt وتوليد وفك الرموز المشفرة (JWT Tokens) كبوابة تحقق لجميع المعابر.
*   **`/server/api/routes.ts`** *(جديد)*: متحكم الـ REST Endpoints المسؤولة عن تسجيل الدخول، تفعيل الحسابات، قراءة/حفظ البيانات، واسترجاع السجلات والنظام الاحتياطي.
*   **`/server.ts`** *(معدل)*: مهندس تشغيل الخادم وتمرير الصلاحيات وقاعدة الترقية الأمنية للـ WebSocket عبر مصادقة الـ Handshake التلقائي.
*   **`/src/App.tsx`** *(معدل)*: تعديل واجهة المستخدم لتمرير الـ JWT Token في مروسات HTTP وفي اتصالات WebSocket، وتحديث واجهتي الدخول والتفعيل لمطابقة الخصائص الآمنة دون أي مساس بمظهر أو وظائف النظام الحالية.

## 💾 2. جداول وفهرسة قاعدة البيانات (Database Schema)

تتألف قاعدة البيانات `inventory.db` من 5 جداول متجانسة محصنة بالمعاملات الذرية لمنع الـ Corruption:

1.  **`users`**: لتسجيل بيانات المستخدمين (المكودين والنشطين):
    *   `code` TEXT PRIMARY KEY (كود أمين المخزن الفريد)
    *   `name` TEXT NOT NULL
    *   `email` TEXT UNIQUE NOT NULL
    *   `phone` TEXT
    *   `role` TEXT NOT NULL (system_admin / program_manager / supervisor / storekeeper)
    *   `password` TEXT NOT NULL (مُشفر بالكامل عبر Bcrypt)
    *   `remember_me` INTEGER DEFAULT 0
    *   `is_precoded` INTEGER DEFAULT 1
    *   `is_registered` INTEGER DEFAULT 0

2.  **`inventory`**: يضم دليل أصناف الكتالوج الرئيسي والكميات الدفترية النشطة:
    *   `id` TEXT PRIMARY KEY (الباركود الرقمي أو المعرف الفريد)
    *   `name` TEXT NOT NULL
    *   `category` TEXT
    *   `bookQty` REAL DEFAULT 0
    *   `unit` TEXT
    *   `previousDiff` REAL DEFAULT 0

3.  **`inventory_snapshots`**: أرشفة سجلات الجرد (الترشيح الدائري بحد أقصى 39 نسخة تاريخية):
    *   `id` INTEGER PRIMARY KEY AUTOINCREMENT
    *   `session_id` TEXT NOT NULL
    *   `date` TEXT NOT NULL
    *   `notes` TEXT
    *   `created_at` TEXT NOT NULL
    *   `snapshot_data` TEXT NOT NULL (التمثيل النصي الكامل لشكل الجلسة التاريخية بهيئة JSON دقيق)

4.  **`settings`**: مخزن الثوابت وضبط النظام وحفظ جلسة الجرد النشطة لحظيًا.

5.  **`audit_logs`**: يسجل جميع التحركات الأمنية والإدارية المهمة:
    *   `id` INTEGER PRIMARY KEY AUTOINCREMENT
    *   `user_code` TEXT (حساب القائم بالعملية)
    *   `action` TEXT NOT NULL (تسجيل الدخول / تعديل الجرد / حذف البيانات / استعادة النسخ الاحتياطية)
    *   `timestamp` TEXT NOT NULL
    *   `ip_address` TEXT
    *   `log_details` TEXT

## 🔒 3. التغييرات والإضافات الأمنية (Security Safeguards)

1.  **مكافحة تخزين البيانات كنص واضح (No Plaintext Password)**: تم تعويم وتأمين كافة كلمات المرور في النظام باستخدام خوارزمية **Bcrypt** بجولة قوة تبلغ 10 جولات تخمير آمنة.
2.  **مصادقة الرمز الآمن (JWT Auths)**: لا يفرغ الخادم أي بيانات في المعابر العامة دون التحقق من ترويسة المصادقة المضمن بها الرمز الصالح (`Authorization: Bearer <JWT_Token>`).
3.  **مصادقة مسار الـ WebSocket**: ترقية معبر الاتصال المباشر محروم عن العموم ولا يتم منح الاتصال بالشبكة للمزامنة الفورية إلا لمن يقدم رمز JWT صالح ضمن متغير المعاينة (`/ws?token=...`). يتم قطع الاتصال فوريًا بالشبكة لمن يفشل في هذا التحقق لحماية السيرفر من هجمات الإغراق.
4.  **التحكم باللوحة وفق الأدوار الإدارية (RBAC Control)**: لا يمكن استخدام صلاحيات تهيئة الروابط، أخذ نسخ احتياطية بنظام الـ Backup، أو استعادة الجلسات التاريخية المؤرشفة إلا لمن يملك صلاحية `system_admin` أو `program_manager`.
5.  **سجل الحركات الشامل (Audit Logs)**: يتم تلقائيًا أرشفة عناوين الـ IP، الطابع الزمني، والرمز التعريفي للمستخدم مع تفصيل العملية التي تم القيام بها.

## 🔄 4. طريقة ترحيل البيانات الحالية من `db.json` إلى SQLite (Zero-Loss Migration)

تم تصميم النظام ليعمل بنمط **الترحيل الآلي الذكي بمجرد التشغيل الأول**:
1.  عند إقلاع الخادم، تقوم طبقة `DatabaseService` بالتحقق من وجود الملف القديم `db.json` على القرص.
2.  في حال وجود الملف وكون قاعدة بيانات SQLite المستهدفة فارغة ومثبتة حديثاً:
    *   يتم قراءة جميع مستخدمي التكويد والتسجيل الفعليين.
    *   **تمر الدورة آليًا على كلمات المرور الحالية**: إن كانت كلمة مرورية واضحة (مثل `123` أو `admin`) يتم فوراً تحويلها وتشفيرها عبر Bcrypt وحفظها مشفرة بشكل لا رجائي وآمن بالكامل.
    *   يتم استيراد كافة أصناف دليل الجرد النشط.
    *   يتم ترحيل وحفظ الـ 3 نسخ المؤرشفة المتواجدة حالياً على موقعك بدقة بالغة داخل جدول الحفظ الدائري `inventory_snapshots`.
    *   يتم تكوين وتأسيس الجلسة النشطة بنفس حالتها التي تركتها تماماً.
3.  بذلك يتم ترحيل موقعك وقاعدتك وتزويدها بالحماية المشددة دون خسارة أي جزئية من العمليات السابقة!

## 🚀 5. خطوات النشر والتثبيت على الـ VPS أو السيرفر الخاص بك

عند رغبتك في رفع التحديثات لبيئة VPS الخاصة بك، يتم النشر ببساطة عبر الخطوات الفنية التالية:

1.  **تهيئة ملف المتغيرات**:
    تأكد من وجود ملف `.env` في المسار الرئيسي لخادمك، وقم بتعيين مفتاح مشفر قوي لتوقيع الـ JWT:
    ```env
    JWT_SECRET=اكتب_هنا_سلسلة_حروف_وارقام_عشوائية_عالية_التأمين
    ```
2.  **تثبيت التحديثات**:
    قم بتشغيل الأمر التالي في مسار المشروع لتثبيت الاعتمادات وتكوين الحزم المغلفة:
    ```bash
    npm install
    ```
3.  **بناء الحزمة النهائية (Production Build)**:
    قم بإجراء بناء الحزمة للتأكد من ربط الواجهات وتوليد الخادم التراكمي المدمج:
    ```bash
    npm run build
    ```
4.  **بدء تشغيل الخادم والترحيل فوريًا**:
    ابدأ تشغيل العملية لتبدأ قاعدة البيانات بتمثيل جداولها وترحيل ملفك `db.json` القديم وبدء الخدمة بمرونة متكاملة:
    ```bash
    npm run start
    ```
    *(ملاحظة: يمكنك استخدام مدراء المهام مثل `pm2` لتشغيل الخادم في الخلفية واستمرارية الخدمة).*
